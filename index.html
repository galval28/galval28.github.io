    The Apocolapse

    The Apocolapse is a handheld gaming device my team and I developed at HackISU fall 2016. It uses right and left pushbuttons as well a 16x2 screen for the UI. It's powered by a mobile power bank attached to the arduino 101 mircocontroler and sotored within the box. The system displays a pre-programed question on the screen and the user answers either yes or no using the button lableded so. The screen then displays a win or loss message depending on if the user pressed the right button, if the user won, the system also vibrates using a vibration motor taken from an old phone. Here is the link to the devpost. I have since up graded the project to store how many times the user has one or lost as well have some questions where the answer is no, as oposed to yes.

    Arduino Music

    I made this porject over winter break '16-'17. The arduino takes the voltage across a potentiometer and maps it to one of five pitch set notes (F-Bflat) which is played though a buzzer. The arduino also reads another input that serves as an on/off function for the music player(not the arduino). There is a 'flip' switch that changes whether the monetary switch generates a note or rest. I will upload pictures later. I plan on expanding this project to be able to save songs to an sd card, as well as play pre-programed songs as read from the sd card. The user would use an LCD screen to see which song they are selecting.

    here is the Git repo with arduino code

    TV monitor

    The arduino based TV monitor was also made over winter break '16-'17. The arduino program simply records how long the Arduino is on for to the SD card that is attached to the arduino. When the Arduino is attached via USB to a TV (wih a usb port) the TV powers the arduino, making it so the arduino is only on when the TV is on, therefore recoding how long the TV is on for. Challenges with this project include: developing a method to update the value stored in the text file on the Sd card thus avoiding long text files witth a tally set for each minute, as well as determining whether to have the values roll over from diferent TV watching sessions. I also had an issue in retriveing the data from the SD card as it would come back in the wrong data type.

    here is the Git repo with arduino code

    Arduino Alarm clock

    I created this project for my ee285 C programing class. I chose this project because I use my phone currently as an alamr clock, and ocasionally this causes issue when my phone dies in the middle of the night. It displays the time on a 16x2 LCD screen. The user is able to set the time and the alarm time using switches and a switch that toggles between the alarm being set and the time being set. If the current time and the alarm set time match the buzzer beeps for a set duration. I ran into issue powering it as there is not enough power from the arduino for both the screen and the buzzer causing the screen to go a little funky.
